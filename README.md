# Tourist RAG Assistant

RAG-ассистент по туристическим объектам: поиск и генерация ответов по корпусу описаний достопримечательностей на русском языке.

## Описание проекта

Этот проект посвящён построению **Retrieval-Augmented Generation (RAG)** пайплайна для ответов на пользовательские вопросы о туристических объектах.  
Система работает на очищенном корпусе достопримечательностей, находит наиболее релевантные объекты и генерирует ответ на русском языке с опорой на найденный контекст.

В рамках проекта были реализованы:

- первичный анализ и очистка датасета;
- построение финального retrieval-корпуса `retr_text_ru`;
- sparse, dense и hybrid retrieval;
- генерация ответов на основе retrieved context;
- оценка retrieval- и generation-качества.

---

## Цель проекта

Цель проекта — построить компактный, но реалистичный **туристический RAG-ассистент** и пройти весь цикл разработки end-to-end:

1. подготовить качественный корпус для поиска;
2. сравнить разные подходы к retrieval;
3. реализовать генерацию ответов на основе найденного контекста;
4. оценить качество полученной системы.

---

## Данные и подготовка корпуса

Проект начинается с исходного датасета туристических объектов.  
Перед использованием данных в retrieval были выполнены несколько шагов предобработки и контроля качества:

- анализ структуры и качества исходных данных;
- проверка и удаление дубликатов;
- нормализация текстовых полей;
- фильтрация шумных и подозрительных записей;
- TF-IDF sanity checks;
- ручной аудит выбросов;
- фильтрация нерелевантных и не-русскоязычных объектов;
- построение итогового текстового поля `retr_text_ru`.

Результатом этой части работы является очищенный финальный корпус `pois_final`, который далее используется во всём retrieval и RAG-пайплайне.

## Архитектура пайплайна

### 1. Retrieval

В проекте сравниваются несколько стратегий поиска:

- **BM25 / lexical retrieval**
- **Dense retrieval** на основе эмбеддингов
- **Hybrid retrieval** через late fusion sparse и dense score

Hybrid retrieval реализован по формуле:

`score = alpha * bm25_norm + (1 - alpha) * dense_norm`

где:
- `bm25_norm` — нормализованный lexical score;
- `dense_norm` — нормализованный semantic score;
- `alpha` — параметр, регулирующий баланс между лексической и семантической составляющей.

---

### 2. Generation

Финальная версия системы использует найденные документы как контекст для русскоязычной LLM.

Пайплайн генерации включает:

- подготовку компактного контекста из найденных документов;
- подачу вопроса пользователя и контекста в модель;
- постобработку ответа;
- формирование финального ответа на русском языке с опорой на retrieved context.

---

### 3. Evaluation

Оценка качества проводится на двух уровнях.

#### Retrieval evaluation
Для retrieval сравниваются разные поисковые стратегии на небольшом evaluation-наборе запросов.

#### Generation evaluation
Для generation используются метрики:

- **Answer Relevancy** — насколько ответ соответствует пользовательскому запросу;
- **Faithfulness (proxy-метрика)** — насколько утверждения в ответе подтверждаются найденным контекстом.

> Важно: текущая реализация faithfulness является упрощённой proxy-метрикой, основанной на выделении утверждений и проверке их лексической близости к retrieved context. Она подходит для внутреннего сравнения версий пайплайна, но не является полноценной LLM-based factual evaluation.

## Основные этапы работы

### Подготовка данных
- анализ исходного датасета;
- очистка и нормализация;
- построение retrieval-текстов;
- формирование финального корпуса.

### Retrieval
- BM25-поиск;
- dense retrieval по эмбеддингам;
- hybrid retrieval через late fusion.

### Generation
- baseline-генерация;
- финальный RAG-пайплайн;
- форматирование контекста;
- постобработка ответа модели.

### Evaluation
- сравнение retrieval-подходов;
- answer relevancy;
- proxy-faithfulness;
- end-to-end оценка на наборе тестовых запросов.

## Пример пользовательского сценария

**Запрос пользователя:**  
`Куда сходить в Ярославле с детьми?`

**Логика работы системы:**
1. из корпуса извлекаются наиболее релевантные туристические объекты;
2. из найденных документов собирается контекст;
3. модель генерирует итоговый ответ на русском языке на основе этого контекста.
